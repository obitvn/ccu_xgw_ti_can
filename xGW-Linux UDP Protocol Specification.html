xGW-Linux UDP Protocol Specification

Project: xGW Robot Communication Interface

Version: 2.0

Date: 2025-12-24

Author: Chu Tiến Thịnh

Transport: Ethernet UDP

Endianness: Little Endian (Standard for ARM & x86)

Alignment: 4-Byte (32-bit) Aligned

1. Overview & Architecture

Giao thức được thiết kế để truyền tải dữ liệu thời gian thực (Real-time) giữa mạch nhúng Gateway (xGW) và máy tính điều khiển (Linux/ROS). Cấu trúc dữ liệu được tối ưu hóa cho việc căn chỉnh bộ nhớ (memory alignment), giúp tăng tốc độ xử lý (Zero-copy parsing) và tránh các lỗi truy cập bộ nhớ trên vi xử lý ARM.

Port PC gửi đi 61904 (0xF1D0)
Port Mạch gửi về 53489 (0xD0F1)

  • Size: 48 Bytes

    • cmd_id = 3: Reboot xGW

4. C/C++ Implementation Header

C

#ifndef XGW_PROTOCOL_H
#define XGW_PROTOCOL_H

#include <stdint.h>

#define PROTOCOL_MAGIC_V2    0x5A5A
#define PROTOCOL_VERSION     0x02

// Message Types
#define MSG_TYPE_MOTOR_STATE 0x01
#define MSG_TYPE_MOTOR_CMD   0x02
#define MSG_TYPE_IMU_STATE   0x03
#define MSG_TYPE_RS485_DATA  0x04
#define MSG_TYPE_XGW_DIAG    0x05
#define MSG_TYPE_XGW_CONFIG  0x06

#pragma pack(push, 1)

// --- Common Header (32 Bytes) ---
typedef struct {
    uint16_t magic;         // 0x5A5A
    uint16_t header_meta;   // Custom Flags/Sub-ID
    uint8_t  version;       // 0x02
    uint8_t  msg_type;      // MSG_TYPE_...
    uint8_t  count;         // Element count
    uint8_t  reserved;      // Padding
    uint32_t sequence;      // Packet counter
    uint32_t payload_len;   // Payload size
    uint64_t timestamp_ns;  // Sync Timestamp
    uint32_t crc32;         // Checksum
    uint32_t reserved_pad;  // Padding -> 32 bytes
} NetHeader_t;

// --- Motor State (20 Bytes) ---
typedef struct {
    uint8_t  motor_id;
    uint8_t  error_code;
    int8_t   pattern;
    uint8_t  reserved;
    float    position;
    float    velocity;
    float    torque;
    float    temp;
} NetMotorState_t;

// --- Motor Command (24 Bytes) ---
typedef struct {
    uint8_t  motor_id;
    uint8_t  mode;
    uint8_t  reserved[2];
    float    position;
    float    velocity;
    float    torque;
    float    kp;
    float    kd;
} NetMotorCmd_t;

// --- IMU State (68 Bytes) ---
typedef struct {
    uint8_t  imu_id;
    uint8_t  reserved;
    int16_t  temp_cdeg;     // Scale: 100
    float    gyro[3];
    float    quat[4];
    float    euler[3];
    float    mag_val[3];
    float    mag_norm[3];
} NetImuState_t;

// --- RS485 Header (Followed by raw data) ---
typedef struct {
    uint8_t  port_id;
    uint8_t  reserved[3];
    uint32_t baud_rate;
} NetRs485Header_t;

// --- Diagnostics (32 Bytes) ---
typedef struct {
    uint32_t uptime_ms;
    uint16_t voltage_in;    // 10mV
    uint16_t current_in;    // 10mA
    int16_t  temp_mcu;      // 0.01C
    int16_t  temp_pwr;      // 0.01C
    uint8_t  cpu_load;
    uint8_t  ram_usage;
    uint8_t  bus_load_can;
    uint8_t  bus_load_485;
    uint32_t status_flags;
    uint8_t  reserved[12];
} NetXgwDiag_t;

// --- Configuration (48 Bytes) ---
typedef struct {
    uint8_t  cmd_id;        // 0:GET, 1:SET, 2:SAVE, 3:BOOT
    uint8_t  reserved[3];
    uint32_t can_baud;
    uint32_t rs485_baud;
    uint32_t ip_addr;
    uint32_t net_mask;
    uint32_t gateway;
    uint16_t udp_port_host;
    uint16_t udp_port_gw;
    uint32_t motor_map_id;
    uint8_t  reserved_block[16];
} NetXgwConfig_t;

#pragma pack(pop)

#endif // XGW_PROTOCOL_H

5. Implementation Notes

  1. Checksum Calculation:
    • Trường crc32 trong Header nên được khởi tạo bằng 0 trước khi tính toán.
    • CRC được tính trên toàn bộ buffer (Header + Payload).
  1. IMU Temperature:
    • Giá trị temp_cdeg là int16_t. Khi Linux nhận: float imu_temp = raw_data->temp_cdeg / 100.0f;
  1. Dynamic Parsing:
    • Bên nhận KHÔNG nên hardcode số lượng motor.
    • Luôn sử dụng header.count để xác định số lần lặp khi parse payload.
    • Luôn kiểm tra header.payload_len so với kích thước thực nhận từ socket recv_len để đảm bảo an toàn bộ nhớ.
  1. Packet Size Examples:
    • 23 Motors Command: 32 (Header) + 23 * 24 = 584 bytes.
    • 55 Motors Command: 32 (Header) + 55 * 24 = 1352 bytes (vẫn nhỏ hơn MTU 1500).
    • 40 Motors Command: 32 (Header) + 40 * 24 = 992 bytes.
    • 1 IMU Feedback: 32 (Header) + 68 = 100 bytes.
    • All sizes are well within standard Ethernet MTU (1500 bytes).
